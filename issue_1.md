# Issue 1

## 背景

当前 Go 后端已经具备控制面基础 CRUD、登录鉴权、策略发布、节点注册、心跳与证书接口的初版实现，但对照 `docs/` 中的功能文档和 `todolist.md`，仍有多项核心能力没有完成，且已有部分文档与实现发生漂移。

本问题单用于固定当前差距，避免后续实现继续在不完整合同上叠加。

## 发现的问题

### 1. 管理员 bootstrap 不完整

- 文档要求：
  - 首次启动自动创建 `admin`
  - 密码随机生成
  - 只展示一次
  - 首次登录强制改密
- 当前实现：
  - 仅自动创建 `admin`
  - 密码固定为 `admin-temp-pass`
  - 没有一次性展示机制
  - 没有 bootstrap 输出流或持久状态管理

影响：

- bootstrap 流程不符合产品定义
- 默认密码固定，存在明显安全风险

### 2. 节点注册缺少审批与信任绑定

- 文档要求：
  - 支持 node enrollment 审批
  - backend 绑定 trust material
  - 节点完成可信接入后再开始 heartbeat/policy sync
- 当前实现：
  - bootstrap token 校验通过后直接创建 node
  - 直接下发 node access token
  - 没有审批状态
  - 没有 trust material/证书/握手材料

影响：

- 节点接入链路与文档不一致
- 多节点信任模型尚未成立

### 3. 策略发布不是按节点编译

- 文档要求：
  - compile snapshot per node
  - 不同节点获取适配自身角色和位置的快照
- 当前实现：
  - 只生成一份全局 payload
  - 将同一 revision 分配给所有节点

影响：

- 无法严格表达多节点差异化可见性
- 不利于后续做节点级链路裁剪和权限边界

### 4. 节点本地策略未持久化，缺少 last-known-good

- 文档要求：
  - 节点本地保存当前 revision
  - 保存 last known good snapshot
  - sync 失败时继续使用最后有效版本
- 当前实现：
  - `policystore` 仅为内存结构
  - 进程退出后状态丢失
  - 没有回滚或恢复策略

影响：

- 节点重启后策略状态不稳定
- 无法满足失败恢复要求

### 5. 路由规则类型未全量完成

- 文档要求：
  - 支持 `domain`
  - 支持 `suffix`
  - 支持 `ip`
  - 支持 `cidr`
  - 支持 `protocol`
- 当前实现：
  - matcher 仅支持 `domain`
  - matcher 仅支持 `domain_suffix`
  - matcher 仅支持 `ip`
  - 未支持 `cidr`
  - 未支持 `protocol`

影响：

- 路由能力未达到产品定义
- 前端若按文档放开配置，将生成后端无法执行的规则

### 6. 证书自动续签未真正实现

- 文档要求：
  - edge node 公网证书自动续签
  - 节点间私有信任证书管理
  - 证书生命周期编排
- 当前实现：
  - 仅暴露 renew API
  - scheduler 仅刷新状态字段
  - 没有实际签发、轮转、存储、分发逻辑

影响：

- 证书能力仍是占位实现
- 无法支撑正式运行

### 7. 节点数据面功能未完全闭环

- 当前已具备：
  - HTTP direct 转发
  - HTTPS `CONNECT` direct 转发
  - chain 模式下的下一跳代理转发
- 尚未完成：
  - WebSocket 明确验证与稳定实现
  - 节点 trust/relay 模块独立化
  - 节点本地 listener 管理
  - 默认策略/拒绝策略精细化

影响：

- 数据面已经从零到一，但不能视为完全成型

### 8. API 文档与实现漂移

- 当前实现：
  - 统一响应结构为 `APIResponse[T]`
  - 控制面与节点接口使用 `Bearer` token
  - enroll 返回 node token
- 当前文档：
  - 仍保留大量裸 JSON 示例
  - `logout` 仍描述为 body 传 `refreshToken`
  - 未同步新的统一响应包裹结构

影响：

- 前端、扩展与后续后端开发会基于错误合同继续推进

### 9. `todolist.md` 与现状未同步

- 当前已有部分 Phase 3 基础代码落地
- 但清单仍全部未勾选
- 同时仍有大量真正未完成项未被分层拆解

影响：

- 项目管理面无法反映真实完成度

## 结论

当前状态应定义为：

- 控制面基础能力已初步实现
- 节点侧运行骨架已初步实现
- 产品定义中的关键能力仍未全量完成
- 文档合同与任务清单需要先校正，再继续深实现

不应判定为“后端完全成型”。
